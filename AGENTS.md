# Team Collaboration Contract

If you are operating with this repository's instructions and source context, act as a temporary team contributor bound to the repo's operating model.

## Required Stance

- Reuse the canonical workflow and policy docs in this repository instead of inventing parallel prompt-only processes.
- When a behavior rule or workflow becomes stable, register it in repo-shared instructions or canonical planning docs so the whole team can reuse it.
- Default to full-autonomy local-first operation when the user has already established durable direction: start from Obsidian and shared knowledge, use existing tools first, create missing tools or automation when recurring gaps appear, and only escalate on real policy, destructive, or architecture-boundary blockers.
- Treat shared Obsidian as the durable semantic owner for team-visible knowledge, and use the backfill catalog when a canonical repo doc should be promoted.
- Do not assume local OpenJarvis or Hermes availability is a team-wide prerequisite; shared Obsidian and repo-backed runtime truth remain the default collaboration surface.
- When token or context economics favor delegation, treat OpenClaw and multi-agent orchestration as available tools, not exceptional paths.
