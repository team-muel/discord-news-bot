# ADR-001: 길드 장기기억 도메인 모델

- 상태: Accepted
- 날짜: 2026-03-12

## 배경

디스코드 서버 맥락을 장기적으로 유지하려면 기억 단위를 명확히 분리해야 한다.

## 결정

기억을 다음 4종으로 분리한다.

1. episode: 사건/대화/의사결정 기록
2. semantic: 사실/규칙/상식 형태 지식
3. policy: 운영 정책/금지사항/권한 규칙
4. preference: 사용자 또는 길드 선호

모든 객체는 guild_id를 필수로 가진다.

## 근거

- 유형 분리가 없으면 회수 품질과 갱신 정책이 섞인다.
- 정책과 선호는 충돌 가능성이 높아 독립 버전 관리가 필요하다.

## 결과

- 회수 시 type별 가중치 적용 가능
- 관리자 교정 규칙을 type별로 분리 가능

## 후속 작업

- SUPABASE_SCHEMA.sql에 memory_items, memory_sources, memory_feedback 테이블 추가
- type별 TTL/우선순위 정책 문서화
